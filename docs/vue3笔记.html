    # vue2 向vue3的迁移

    • 异步组件

      
const asyncCmp = defineAsyncComponent(() => import("xxx"))

或者

const asyncCmp = defineAsyncComponent({
	loader:() => import("xxx"),
	delay:200,
	timeout:3000,
	errorComponent:ErrorComponent,
	loadingComponent:LoadingComponent
})

    • $children 被移除

      vue3中访问子实例建议采用 $refs

    • 自定义指令

      Vue2中:
bind - 指令绑定到元素后调用。只调用一次。
inserted - 元素插入父 DOM 后调用。
update - 当元素更新，但子元素尚未更新时，将调用此钩子。
componentUpdated - 一旦组件和子级被更新，就会调用这个钩子。
unbind - 一旦指令被移除，就会调用这个钩子。也只调用一次。

Vue3中：
created - 新增！在元素的 attribute 或事件监听器被应用之前调用。
bind → beforeMount
inserted → mounted
beforeUpdate：新增！在元素本身被更新之前调用，与组件的生命周期钩子十分相似。
update → 移除！该钩子与 updated 有太多相似之处，因此它是多余的。请改用 updated。
componentUpdated → updated
beforeUnmount：新增！与组件的生命周期钩子类似，它将在元素被卸载之前调用。
unbind -> unmounted

const MyDirective = {
  created(el, binding, vnode, prevVnode) {}, // 新增
  beforeMount() {},
  mounted() {},
  beforeUpdate() {}, // 新增
  updated() {},
  beforeUnmount() {}, // 新增
  unmounted() {}
}

不同点： 
vue2 获取 组件实例： 
bind(el, binding, vnode) {
  const vm = vnode.context
}

vue3 获取组件实例：
mounted(el, binding, vnode) {
  const vm = binding.instance
}

    • Vue3中data只能声明为function

    • 子组件发送的事件要声明在emits当中

      Vue3移除了 .native 修饰符 (opens new window)。任何未在 emits 中声明的事件监听器都会被算入组件的 $attrs，并将默认绑定到组件的根节点上。

    • 移除$listeners

      之前的v-on="$liseners"在vue3中不在适用
// 父组件传过来的所有事件全部在$attrs中
统一采用 v-bind="$attrs"

    • 移除了.native

      ​ 任何未在emits里面声明的事件监听器都会被算入组件的$attrs,并将默认绑定到根节点上

    • 移除了$on,$off,$once

    • Vue.prototype 替换为 config.globalProperties

    • Vue.extend移除

      vue3中使用
const app = createApp({});
app.mount("#xxx")
替代

    • Vue.observable 用Vue.reactive替换

    • v-if、v-else、v-else-if的各项分支key将不再是必须的，因为现在Vue会自动生成唯一的key

    • <template v-for>

      在vue2中 template标签不能拥有key，不过可以对其每个子节点设置key

在vue3中key应该直接设置在template上

    • v-for 和 v-if的优先级问题

      在vue2中 v-for的优先级高于v-if

在vue3中 v-if的优先级高于v-for的

    # setup

    ​ 在setup中无法调用this，setup的执行时机在beforeCreate和created之间，此时data、computed、methods还没有被解析，此时组件实例未实例化成功。

    # prop声明

    defineProps 用于声明从父组件传入的prop
withDefaults 用来声明props的默认值
defineEmits 用于声明子组件导出的事件


在ts中声明widthDefaults
interface Props{
    xxx:xxx
}

withDefaults(defineProps<Props>,{
              xxx:xxx
              })


在ts中声明defineEmits
// e 为emit的名字，第二个参数的key可以随意定义
const emit = defineEmits<{
  (e:'change',id:number):void,
  (e:'update',value:string):void
}>();

    defineProps在<script setup>中自动可用，无需导入

    需在.eslintrc.js文件中【globals】下配置【defineProps: true】

    # 解构reactive

    vue3中如果直接使用解构赋值解构通过reactive声明得响应式变量，那么解构出来的值就不会是响应式的了。

    解决方案：使用toRefs解构reactive

    const state = reactive({
    name:"zhagnsan",
    age:21
})

const {name,age} = toRefs(state);

    # 父组件使用子组件的数据

    在script-setup模式下，所有数据只是默认return给template使用，不会暴漏到组件外，所以父组件的ref是获取不到子组件的数据

    如果父组件想要拿到子组件的数据可以通过defineExpose来完成

    defineExpose({
    ...toRefs(state),
    changeName(){
        state.name = "Tom"
    }
})

    # 使用插槽

    <template>
	<slot/>
	<slot name='title'/>
	<slot name='footer'/>
</template>
<script setup>
    import {useSlots,reactive} from "vue";
    const slots = useSlots();

    const titleSlot = reactive(slots.title && slots.title().length)
</script>

    # 获取上下文

    main.js中 
app.config.globalProperties = {
    a:1,b:2
}
组件内

import {getCurrentInstance} from 'vue';

const {proxy,ctx} = getCurrentInstance();

proxy.a // 1
ctx.a // 1

    注意： 上述proxy、ctx都能够获取到组件的上下文

    ctx 只能在开发环境下使用，生产环境下ctx将访问不到

    proxy 在开发环境和生产环境下都能够使用

    # script-setup中对await的支持

    script-setup 中可以不用配合async去使用 await 了， 下面的情况下组件的setup会自动变成async-setup

    const post = await fetch("/api").then(() => {})

    # 定义组件的name

		name:"ComponentName"
	}
</script>
<script setup>
    xxx
</script>

    # 常用API

    • ref

      // 接受内部值并返回一个响应式且可变的ref对象。
// ref对象仅有一个 `.value`属性，指向该内部值

    • reactive

      // 返回对象的响应式副本
// 响应式转换是“深层”的——它影响所有嵌套 property

    • toRef

      // 将reactive定义的对象中的某个属性转成ref

const state = reactive({
    foo:1
})

const fooRef = toRef(state,'foo')

    • toRefs

      // 将响应式变量转成 普通对象，普通对象里面的每一个属性都是一个ref

const state = reactive({
    foo:1,
    bar:2
})

const stateRefs = toRefs(state);

stateRefs.foo.value++;

const value = isRef(refValue) ? refValue.value : refValue;
