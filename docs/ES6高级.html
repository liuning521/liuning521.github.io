    let f = fn();
    next();
    function next(data){
        const res = f.next();

            res.value.then(res => {

            })
        }
    }
}

    # 同步、异步

    同步和异步是一种消息通知机制

    • 同步阻塞：做一件事，没有拿到结果之前就一直在这等着，一直等到有结果了，再去做下边的事。A调用B，B处理获得结果，才返回给A。A在这个过程中，一直等待B的处理结果，没有拿到结果之前，需要A一直等待和确认调用结果是否返回，拿到结果然后继续往下执行。

    • 异步非阻塞：做一件事情，不用等待事情的结果，然后就去忙别的了，有了结果在通过状态来告诉我，获取通过回调函数来处理。A调用B，无需等待B的结果，B通过状态，通知等来通知A或回调函数处理。
