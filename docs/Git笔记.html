    # Git基础命令

    // 在当前目录下生成一个 .git文件夹用于存放git操作历史记录
// 初始化创建仓库
git init

//  查看当前仓库所在目录的文件状态
git status  // 打印文件状态 （未追踪、已修改、已暂存）

git status -s = git status --short // 简化文件状态打印内容



// 将文件加入到追踪
git add filename

// 将文件添加到本地仓库
git commit -m "xxxx"

// 打印当前所在分支的已提交的日志
git log 

// 删除git区域中记录的文件，并且不保留在工作目录中
git rm <file> 
    
// 强制删除
git rm -f(force) <file>
    
// 删除Git仓库中保留工作目录中的文件,不删除工作目录中的文件
git rm --cache <file>
    
// 移动文件(其实做的是重命名操作)
git mv <originPath> <newPath>

// 上述操作等同于
mv <originPath> <newPath>
git rm <originPath>
git add <newPath>
    
git diff // 查看当前文件修改的内容 对比工作目录和暂存内容的差别 

git diff --stage // 查看当前暂存的和已经提交的内容的差别

git log // 查看提交日志信息

git log -p // 查看最近一条提交的详细信息

git log -n // 查看最近n条信息

git log --stat // 列出所有被修改的文件，以及简略的统计信息

    ::: tips

    D 标识删除的文件

    R 代表重命名

    :::

    # Git 操作过程

    文件的三种状态： 已修改、已暂存、已提交

    未被git追踪的文件属于未追踪状态，因为还没被git记录追踪，不在git文件的三种状态之中

    三个工作区域： 工作目录、暂存区域、Git仓库

    Git工作流程：

    1. 在工作目录中修改文件
    2. 暂存文件
    3. 提交更新，找到暂存区文件

    # Git拓展命令

    可以通过 git add .  或者 git add -a 添加所有改动文件以及未追踪的文件

可以通过 git commit -m "描述"  合并提交和描述，一步完成操作

git commit -a -m "描述" 从工作目录提交到暂存区后，直接提交

    如果git命令行出现了文件名乱码，可以在git命令行执行

    git config --global core.quotepath false

    如果是编辑描述乱码

    在命令行右键进入setting - 找到environment - 添加 set LANG=zh_CN.UTF-8

    # 分支操作

    git分支本质上仅仅是指向提交对象的可变指针

    git默认分支名称：master，它会在每一次的提交中自动前移

    git branch # 查看所有分支

git branch <branchName> # 创建一个新分支

git branch -d <branchName> # 删除分支

git branch -D <branchName> # 强制删除分支

git checkout <branchName> # 切换分支

git checkout -b <分支名称> # 创建并选中分支


git log --decorate --oneline # 以简短的方式打印当前分支上的提交记录

    HEAD是一个特殊的指针，指向当前所在的分支

    --decorate标记让git log显示指向这个提交的所有引用（比如说分支、标签等）

    --oneline标记把每一个提交压缩到了一行中。它默认只显示提交ID和提交信息的第一行。

    --all 打印所有分支的信息

    --graph 可以在打印的前面看到一个树形的图， 选项绘制一个ASCII图像来展示提交历史的分支结构 ， 它经常和 --oneline和 --decorate两个选项一起使用，这样会更容易查看哪个提交属于哪个分支

    HEAD指向的分支无法删除

    从未参与过合并的分支也无法删除，需要通过git branch -D <分支名>进行删除

    # 分支合并

    git checkout master # 选择主分支

git merge <目标分支> # 将目标分支的内容合并到当前分支

git merge --abort # 合并冲突的时候 取消合并

    # 快速前移

    当master和branch分支没有形成分叉，依旧是处于一条路径线，当HEAD落后于要合并的分支，将会形成快速前移。快速前移之后使用git log打印不会体现出进行了合并操作

    git merge --no-ff -m '这是一个合并操作' <branchName>  
# --no-ff 是禁止快速前移

    # 冲突出现的原因

    合并分支的时候如果两次提交时祖先级的关系，不会冲突，如果不是祖先级的关系则会冲突

    # 取消大合集

    # 取消对文件的追踪
git rm --cached <filename> 

# 撤销上一次的修改（必须是还没有加入到暂存区）
git checkout -- <filename>

# 取消掉已经添加到暂存区的文件
git reset HEAD <filename>

# 撤销掉commit提交 写几个‘^’ 表示撤销最近几次的commit
git reset "HEAD^"

# 撤销掉最近n条commit提交
git reset HEAD~n


# 替换掉上一次commit的信息,必须紧接着上一次提交执行
git commit -m <msg> --amend


# 查看历史提交记录，会打印出每次历史提交记录以及提交的ID
git reflog

# 找回撤销的提交
git reset <gitHash>

    # 存储

    当在一个分支中向暂存区添加内容，切换到另一个分支时，提交commit会带上另一个分支中的暂存内容。

    git stash # 存储暂存区及工作目录未修改文件,默认不存储未追踪的文件

git stash -u # 在上面的基础上带上未追踪文件 

git stash list # 打印所有存储内容

git stash apply <stashName> # 将存储内容重新应用（默认不保留已暂存内容） stashName 为git stash list 列表中的标号

git stash apply --index # 将原暂存依旧以暂存进行取出

git stash drop <stashName> # 移除存储

    # rebase 变基

    和merge一样用于分支合并，将一个分支的内容都移至另一个分支上。

    工作流程：

    • 首先找到两个分支的共同祖先
    • 然后对比当前分支与祖先的历次提交，进行提取相应修改，并保存为临时文件，将当前分支指向目标基底，最后将之前存为临时文件的修改依序应用。
    
# 执行该命令前，要先将分支切换到要被合并的分支下
# 以branchName分支为基，合并当前所在分支
git rebase <branchName>

#执行完上述后 代码还没有合并，还想将分支切换到主分支，执行git merge命令

    rebase 合并会保留分支的每一次提交的过程

    git merge 注重结果，不会保留分支的提交过程

    # 添加远程仓库并提交

    git remote add origin <远程仓库地址>

# 将暂存区的内容推送到远程仓库 -u 代表后期的所有提交都添加到远程仓库 分支名称这
git push -u origin <分支名称>

git clone <项目路径>

# 拉取
git pull 

# 远程仓库克隆分支
git checkout -b <本地分支名称> <远程分支名称>一般为 (origin/远程分支名称)

# 删除远程分支，先删除本地分支
git push origin :分支名称

# 推送标签到远程仓库
git push origin <标签名称>

# 上传所有标签
git push origin --tags

# 删除远程仓库标签
git push origin :refs/tags/<标签名称>

    拉取下来的分支默认是master

    # tag标签

    软件的 “xx版本” 就是所谓的标签，该标签指向一个commit对象，单commit对象中的hash值太长，并且没有规律，所以我们使用标签的方式，进行版本标注。

    git tag <tagName> #(默认在最新的commit提交上)
eg:  git tag v1.0
git tag #查看标签

# 给指定 commit 添加标签（已经提交过的commit）
git tag <tagName> <targetCommitHash>

# 添加带有说明的标签
git tag -a <tagName> -m <说明信息> <commitHash>

#通过下面的命令查看版本号的说明
git show <tagName>

# 删除标签
git tag -d <tagName>
