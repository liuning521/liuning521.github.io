    # Linux服务器上装Nginx

    • 将Nginx的安装包拖到linux服务器上
    • 然后进入Nginx目录，执行 ./configure 代表检查当前环境是否满足安装，不满足会报错
    • Nginx是使用c写的，首先需要安装 gcc ， yum install gcc就可以安装完成了
    • 下载一个 pcre (opens new window)拖到服务器上，然后进入改目录执行 ./configure
    • pcre需要gcc-c++ 的支持，所以需要安装 gcc-c++ 才能安装 pcre 成功， 执行 yum install gcc-c++ 即可安装成功
    • gcc-c++ 安装成功后，回到 pcre的根目录，先执行./configure 在执行make install 即安装命令即可安装 pcre成功
    • 回到nginx目录，继续执行./configure 提示缺少zlib
    • 下载zlib (opens new window) ，拖到服务器上，然后进入zlib根目录执行 ./configure 在make install 即可安装zlib成功
    • 回到 nginx目录，执行 ./configure 然后在执行 make install 即可安装nginx成功
    • nginx 安装成功后，nginx程序会在 /usr/local 目录下 有一个nginx 目录， nginx 目录下有一个sbin ，进入sbin目录 执行 ./nginx 即可启动nginx成功
    • nginx 重启命令 nginx -s reload nginx 停止命令 nginx -s stop

    # 使用fetch获取本地Excel信息一直获取不到得问题

    fetch请求获取文件，请求状态一直是200，但一直拿不到文件信息，说明请求得文件路径错了
另外就是
静态资源不能放到
src里面，是因为src里面的文件名在webpack打包编译的时候文件名变了，而打包不会更改fetch里面引用的名称，这就导致了fetch请求的路径对应的文件不存在，所以才导致请求不到本地文件信息

    静态文件要放到public里面，在public里面得东西不会被打包进去，文件名就不会被修改

    # 将base64格式的数据下载下来

    base64格式的数据格式一般是： data:[类型];base64,[文件base64编码]
第一步先拿到文件base64编码

const data = base64Str.split(',')[1];

第二步 对文件base64格式的编码进行解码
const decodeData = window.atob(data);

第三步 将其转换成 Uint8Array 数组

const ia = new Uint8Array(decodeData.length);
for(let i= 0; i< ia.length;i++){
    // 拿到每一位编码对应的unicode值
    ia[i] = decodeData.chartCodeAt(i);
}

const blob = new Blob([ia],{type:"application/octet-stream"});

最后就是把 blob文件下载下来

    # 把图片转成base64格式

    从图片源获取图片将其转成base64格式

    getImage(){
   const imagePath = 'static/123.jpg';
   const img = new Image();
    // 如果文件是跨域获取的，需要把下面的注释打开
    // img.crossOrigin="anonymous";
    img.onload = function(evt){
        // 获取文件拓展名
        const ext = getImgExt(imagePath)
        const data = getBase64String(img,ext);
    }
}

getBase64String(image,ext){
    const canvas = document.createElement("canvas");
    canvas.width = image.width;
    canvas.height= image.height;
    const ctx = canvas.getContext('2d');
    ctx.drawImage(image,0,0,image.width,image,height);
    return canvas.toDataURL('image/'+ext);
}

getImageExt(path){
    const idx = path.lastIndexOf(".");
    return path.slice(idx+1);
}

    注意获取本地文件一定要放到 public/static文件夹下，否则可能会有问题

    # Vue性能优化

    • 变量本地化提升性能

      在vue中每次执行this.xxx 都会触发依赖收集中的getter方法，然后执行依赖收集的相关代码，变量使用的次数越多，性能越差
可以通过 let {xxx} = this; 将变量缓存到本地的方式去更新变量

    • 事件销毁

      Vue组件销毁时，会自动解绑它的全部指令及事件监听器，但是仅限于组件本身的事件，对于定时器、addEventListener注册的监听器等，就需要在组建销毁的生命周期钩子中手动销毁或解绑，以避免内存泄漏

      # Proxy 和 Object.defineProperty的区别

      • Object.defineProperty只能监听到属性的读写，Proxy除了可以监听属性的读、写、删除、调用等13中操作
      • Object.defineProperty不能监听数组的变化，Proxy可以
      • Proxy是以非入侵的方式监管了对象的读写，而defineProperty需要按特定的方式定义对象的属性

    在同步代码中注册的微任务，会在第一个异步宏任务之前执行。微任务也是异步任务，所以会放在异步任务队列中执行。之后的宏任务中的微任务，会在下一个宏任务执行之前执行完毕，当此任务中的宏任务会自动追加到任务队列尾等待执行。

    # JavaScript中代码执行流程

    • 默认的同步代码按照顺序从上到下，从左到右运行，运行过程中注册本次的微任务和后续的宏任务
    • 执行本次同步代码中注册的微任务，并向任务队列中注册为任务中包含的宏任务和微任务
    • 将下一个宏任务开始前的所有微任务执行完毕
    • 执行最先进入队列的宏任务，并注册当次的微任务和后续的宏任务，宏任务会按照当前任务队列的队尾继续向下排列
