    正则表达式

    liuning

    # 创建正则表达式

    1. 字面量创建

      const reg = /\d+/g;
const str = "abcdefg123123fef";
str.match(reg); // 返回所有满足条件的结果数组

    2. 构造函数创建

      // 构造函数创建需要转义
const reg = new RegExp("\\d+","g")

      通过字面量创建正则表达式 不能用变量

      通过构造函数创建正则表达式可以使用变量

    # 正则匹配方法

    # 正则对象底下的方法

    1. test: 看字符串是否符合正则表达式标准

      const reg = /\d+/g;
reg.test("234abd123");

    2. exec: 匹配到满足条件的正则表达式，一次只返回一个满足条件的表达式，可执行多次即每一次会基于上一次的结果中的lastIndex去继续往后匹配

      const reg = /\d+/g;
const str = "234abd123";
reg.exec(str) // 234
reg.exec(str) // 123

    # 字符串方法

    1. split:可以基于正则表达式切割字符串

      const str = "abc123fda1234";
const arr = str.split(/\d+/);

    2. search:返回满足条件的正则表达式第一次出现的索引，会忽略全局匹配,不存在会返回-1

      const str = "abc123fda1234";
str.search(/e/)

    3. match:匹配到满足正则条件的值

      const reg= /\d+/g
const str = "abc123fda1234";
str.match(reg)

    4. replace:替换掉满足正则条件的字符串

      const reg= /\d+/g
const str = "abc123fda1234";
str.replace(reg,'');
str.replace(reg,function(arg，...group){
    console.log(arg); // 正则匹配到的内容
    console.log("group"); // 分组的内容
    return '';
})

    # 元字符： 有特殊函数非字母字符

    1. 字符相关

      \w: 数字、字母、下划线

      \W: 非 数字、字母、下划线

      \d: 数字

      \D：非数字

      \s：匹配空格

      \S： 非空格

      . : 非 \n \r \u2028 \u2029 (换行、回车、段落结束符、行结束符)

    2. 数量相关

      {n}： 一种语义表示 匹配的数量 另一种语义n表示unicode编码（开启u匹配模式的情况下）

      const str = "abcccdefg";
// 只匹配 出现三次的c
const reg = /bc{3}d/g
// 匹配出现三到四次的c
const reg = /bc{3,4}d/g
// 最少出现一次c
const reg = /bc{1，}d/g
// 默认为贪婪匹配，尽可能多的去匹配
const reg = /\d{2,4}/g
const str = "123456789"
str.match(reg) // ["1234","5678"]
// 开启惰性匹配或非贪婪匹配
const reg = /\d{2,4}?/g
str.match(reg) // ["12","34","56","78"]

      ?: ==> {0,1} 0次或一次

      '+': == > {1,} 一次以上

      *: ==> {0,} 0次以上

    3. 位置相关

      ^: 开始位置

      const str = "abc";
const reg = /^/g // 表示最开始的位置
str.replace(reg,"*") // "*abc"

      $ ：结束位置

      const str = "abc";
const reg = /$/g // 表示结尾的位置
str.replace(reg,"*") // "abc*"

      \b： 边界符

      const str = "this is a book";
const reg = /\bis\b/g
str.match(reg) // ['is']

      \B: 非边界

      const str = "this is a book";
const reg = /\B\w{2}\b/g // 左边没有边界，右边有边界的两个字符
str.match(reg) // ['is',"ok"]

      边界： 非\w 的都是边界

    4. 括号相关

      () : 分组 跟数学中的() 差不多

      const str = "abababccccddddefg";
// 匹配ab出现的次数
const reg = /(ab){3}/g;
console.log(reg.test(str));
// 匹配到具体的年月日
const str = "2021-11-28";
const reg = /(\d{4})-(\d{2})-(\d{2})/
str.match(reg); // ["2021-11-28","2021","11","28"]
// 或者通过全局属性去拿
RegExp.$1  ==> 2021
RegExp.$2 ==> 11
RegExp.$3 ==> 28
// 替换
const str = "2021-11-28";
const reg = /(\d{4})-(\d{2})-(\d{2})/;

str.replace(reg,"$2/$3/$1")
或者
str.replace(reg,function(arg,year,month,date){
    // year month date 为分组的内容即 $1,$2,$3
})

// 反向引用
const name = "news-container-nav";
 // 假设"news_container_nav" 也满足条件
const reg = /\w{4}(-|_)\w{9}(-|_)\w{3}/
reg.test(name) // true
// 在假设 前面匹配到的是 - 后面也必须是-不能是_
// 这种情况就可以使用反向引用 \n  n为具体数字
// \n 代表前面分组的第几个 
const reg = /\w{4}(-|_)\w{9}(\1)\w{3}/

      []: 字符集合

      const str = "My name is LiLei";
// 匹配LiLei  第二个L 可以是大写也可以是小写
const reg = /Li[Ll]ei/g
// 中括号表示字符集合 只要是[] 里面的一个就行

// 匹配数字
const reg = [0-9];
// 匹配字母
const reg = [A-z]
// 表示否定 [] 中 ^ 表示否定
const reg = [^0-9];

      注意： 想要在[] 中使用- 的话，-两端的ASCII码必须是连续的才可以用

      \d 等同于 [0-9]

      \w 等用于 [a-zA-Z0-9_]

      {n}: 表示 匹配的数量 另一种语义n表示unicode编码

    # 匹配模式

    • g：全局匹配

    • i： 忽略大小写

    • m：多行模式

      const str = `abc
efg
hij
`;
const reg = /^\w/g;
str.replace(reg,"*") // 这样只会把a变成*
// 如果想把每一行都换成 *的话
const reg = /^\w/gm;
str.replace(reg,"*") // 这样只会把每一行的开头字母变成*

    • s： 让"."能够匹配到换行

      const str = `abc
efg`;
const reg = /^a.*g$/g;
reg.test(str) ==> false
const reg = /^a.*g$/gs;
reg.test(str) ==> true

    • u: 匹配unicode编码

      const str = "a";
const reg = /\u{61}/gu
reg.test(str) ==> true

    • y: 粘性模式，默认为非粘性模式

      const str ="12345abcdefg67890";
const reg = /\d/g;
reg.exec(str) ==> 1
reg.exec(str) ==> 2
reg.exec(str) ==> 3
reg.exec(str) ==> 4
reg.exec(str) ==> 5
reg.exec(str) ==> 6
... 

如果开启粘性模式后，所匹配的字符必须是连续的
所以下面的正则表达式 匹配到5后就执行完毕了
const reg = /\d/gy;

    # 命名分组及零宽断言

    # 命名分组

    const str = "2021-11-28";
// 在分组里面使用 ?<groupName> 的形式去自定义分组名称
const reg = /(?<year>\d{4})-(?<month>\d{2})-(?<date>\d{2})/;

str.match(reg)中的 groups字段中会看到分组名称对应的值

    # 零宽断言

    正向肯定零宽断言： 匹配后面满足条件的

    const str = "iphone3iphone4iphone5iphone6iphone7iphonenumber";
// 要求使用正则表达式把iphone换成 ‘苹果’

const reg=  /iphone\d/g
str.replace(reg,"苹果") // 这种情况下会把iphone 和iphone后面的数字都给替换成 苹果不满足要求
可以通过 正向肯定零宽断言进行处理
const reg=  /iphone(?=(?:\d))/g
str.replace(reg,"苹果")// 这样只会把iphone后面带数字的所有 iphone 换成 苹果而不会替换其后面的数字

    ​ "?:" 表示匹配冒号后面的内容但不获取匹配结果，不进行存储供以后使用

    正向否定零宽断言 只需要把 上面的 = 换成 ! 就行了

    const reg=  /iphone(?!\d/)g
const str = "iphone3iphone4iphone5iphone6iphone7iphonenumber";
str.replace(reg,"苹果") // 这种情况下只会把 iphonenumber 中的iphone 替换成苹果

    负向肯定零宽断言： 匹配前面满足条件的

    const str = "10px20px30pxipx";
// 把px换成 像素
const reg = /(?<=\d+)px/;
str.replace(reg,"像素")

    负向否定零宽断言

    const str = "10px20px30pxipx";
// 把px换成 像素
const reg = /(?<!\d+)px/;
str.replace(reg,"像素")

    # 数字分割

    const str = "100000000000000000000.00"
const reg = /\B(?=(?:\d{3})+(?:\.\d*)*$)/g
str.replace(reg,",");
