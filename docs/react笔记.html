    # JSX

    JSX:JavaScript + XML

    jsx语法

    • className 代替 class 添加class类
    • JSX中的属性和方法都是驼峰式的， onclick会变成onClick
    • 标签必须以斜杠结尾 例如：<img/>
    • 变量、函数和属性可以通过单花括号嵌入 即 { name }

class Table extends Component{
    render(){
        const {columns} = this.props;
    }
    return (
    	<table>
        	<TableBody columns={columns} />
    	</table>
    )
}


const TableBody = (props) => {
  const rows = props.characterData.map((row, index) => {
    return (
      <tr key={index}>
        <td>{row.name}</td>
        <td>{row.job}</td>
      </tr>
    )
  })

  return <tbody>{rows}</tbody>
}

    一个类组件必须包含render()，并且return 只能返回一个父元素

    # React基础

    React DOM 会将元素和他的子元素与他们之前得状态进行比较并只会进行必要得更新来使DOM达到预期的状态

    构造函数是唯一可以给this.state赋值的地方

    # 自定义组件

    • React会将以小写字母开头的组件视为原生DOM标签，所以自定义组件的名称必须以大写字母开头。
    • 所有的React组件都必须想纯函数一样保护他们的props不被更改

    # 组件生命周期

    • componentDidMount 组件已经被渲染到DOM中后执行
    • componentWillUnmount

    # 渲染

    key只是在兄弟节点之间必须唯一，数组元素中使用的key在其兄弟节点之间应该是独一无二的。

    # 事件bind this

    在constructor中 修改方法的this指向，因为类中定义的方法里面this默认指向undefined

    contructor(){
    this.handleClick = this.handleClick.bind(this);
}

// 这种定义，需要在contructor中改变this指向 
handleClick(){
    this.setState({
        name:"zhangsan"
    })
}

// 这种定义，不用改变this指向
handleClick = () => {
    this.setState({
        name:"lisi"
    })
}

// 绑定的方法最后可加一个参数表示event

    React中的Event其实是React封装的，__proto__.constructor是SynthetickEvent 模拟出来DOM事件所有能力

    event.target 指向当前元素

    event.currentTarget 指向当前元素

    获取原生Event可以通过event.nativeEvent

    event.nativeEvent.target 指向当前元素

    event.nativeEvent.currentTarget 指向document

    1. event是SyntheticEvent模拟出来DOM事件所有能力
2. event.nativeEvent是原生事件对象
3. 所有的事件，都被挂载到document上
4. 和DOM事件不一样，和Vue事件也不一样
